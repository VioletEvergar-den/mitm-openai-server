


# 命令行实现
- 使用cobra组织命令行参数，至少应该多个子命令
- 至少应该有api子命令，能够启动api服务
- api服务支持从命令行配置认证，如果用户没有配置的话，则不配置认证，表示没有认证，传递任何认证都能使用
- 对于每个子命令及其参数，请充分测试可用


# 前端界面要求
- 对于捕获到的请求，你应该有一个前端界面能够展示
- 这个前端界面应该有认证，比如固定的用户名和随机的密码，启动的时候把凭证信息打印到控制台上
- 用户应该登录之后才能看到界面，先是登录，登录成功能看到列表页
- 列表页就是捕获到的所有的请求数据 
- 除了列表页，还应该有一个界面，引导用户如何在可以配置openapi的地方，配置上当前这个服务
- 因为登录之后也有多个页面，所以应该有一个导航栏，导航栏上有不同的菜单，可以跳转到不同的页面
